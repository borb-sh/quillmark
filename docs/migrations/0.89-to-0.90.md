# 0.89.0 → 0.90.0 — engine-free `Quill`, split WASM, one core type

A structural release that decouples *holding a quill* from *rendering one*, and
splits the WASM package so an editor can load and validate without downloading
Typst.

Three connected changes:

1. **`Quill` is engine-free data.** It no longer holds a backend. The
   `Quillmark` engine becomes a backend registry + render dispatcher; rendering
   and backend-dependent capability move onto it. The backend-existence check
   moves from load time to render time.
2. **The WASM package ships two builds.** A Typst-less **core**
   (`@quillmark/wasm/core`, ~0.66 MB gzip) and a Typst-backed **render**
   superset (the root `@quillmark/wasm` import, ~8 MB).
3. **`QuillSource` and `Quill` collapse into one core type, `Quill`** (Rust
   surface only — bindings already hid `QuillSource`).

## WASM / JS

### Construction: the engine no longer loads quills

`engine.quill(tree)` is removed. Construct a quill directly — no engine needed:

```diff
- import type { Quill } from '@quillmark/wasm';
+ import { Quill } from '@quillmark/wasm';   // value import — needed for Quill.fromTree

- const engine = new Quillmark();
- const quill = engine.quill(tree);
+ const quill = Quill.fromTree(tree);
```

Note: `Quill` must be a **value import** (not `import type { Quill }`). Projects
using `verbatimModuleSyntax` or aggressive type-stripping will hit a compile
error on the static call site until the import is changed.

`Quill.fromTree` accepts the same `Map<string, Uint8Array>` (or plain object) as
before. It is the **full parse-and-validate step** — it throws on a malformed or
schema-invalid tree. A `Quill` is now portable, validated data: `backendId`,
`metadata`, `schema`, `blueprint`, `validate`, `seedDocument` / `seedMain` /
`seedCard` all work without an engine.

**Testing note:** Code that previously mocked `engine.quill(tree)` to return a
fake quill must now stub the static constructor directly. Import `Quill` from
the build your code under test imports — `@quillmark/wasm/core` for quiver
consumers and schema-only code, `@quillmark/wasm` for render-path code:

```ts
// Quiver consumer / core-only code (e.g. getQuill tests)
import { vi } from 'vitest';
import { Quill } from '@quillmark/wasm/core';

const spy = vi.spyOn(Quill, 'fromTree').mockImplementation((tree) => {
  return fakeQuill as unknown as Quill;
});

// Render-path code (e.g. preview tests that call engine.render)
import { Quill } from '@quillmark/wasm';
const spy = vi.spyOn(Quill, 'fromTree').mockImplementation(...);
```

The spy must target the same module the code under test imports — spying on
the render `Quill` has no effect on a code path that calls `Quill.fromTree`
via the core build, and vice-versa.

### Rendering and capability move onto the engine

`render`, `open`, and capability are no longer methods on `Quill`; they live on
the `Quillmark` engine and take the quill as the first argument:

```diff
- const result  = quill.render(doc, opts);
- const session = quill.open(doc);
- const canPreview = quill.supportsCanvas;
+ const engine  = new Quillmark();      // render build only
+ const result  = engine.render(quill, doc, opts);
+ const session = engine.open(quill, doc);
+ const canPreview = engine.supportsCanvas(quill);
```

`RenderSession` is unchanged: `session.render(opts)`, `pageSize`, `paint`,
`pageCount`, `backendId`, `supportsCanvas`, `warnings`.

### `supportedFormats` left `metadata`

`Quill.metadata` is now pure config (identity only). The backend's output
formats are a resolved-backend capability — read them from the engine:

```diff
- const formats = quill.metadata.supportedFormats;
+ const formats = engine.supportedFormats(quill);   // OutputFormat[]
```

The `[key: string]: unknown` index signature has been **removed** from
`QuillMetadata`. Code that reads `quill.metadata.supportedFormats` now produces
a compile-time error (`Property 'supportedFormats' does not exist on type
'QuillMetadata'`) instead of silently returning `undefined` at runtime.

### Backend errors surface at render time, not load time

`Quill.fromTree` now succeeds even when the declared backend isn't available
(the core build has no backend at all). The `UnsupportedBackend` error moves to
the first engine call — `open` / `render` / `supportedFormats` /
`supportsCanvas`.

### Two entry points

```js
// Editor: schema + validation only, no Typst (~0.66 MB gz)
import { Document, Quill } from '@quillmark/wasm/core';

// Preview / export: superset with the engine + canvas (~8 MB gz)
import { Document, Quill, Quillmark } from '@quillmark/wasm';
```

The root `@quillmark/wasm` *is* the render build (there is no separate
`/render` subpath). The two modules have **separate linear memories** — a
`Quill` / `Document` from core is not a usable handle in render. Cross the
boundary as data: re-feed the tree to `Quill.fromTree` and round-trip the
document through `doc.toJson()` / `Document.fromJson()`.

## Rust

### Constructors and rendering

```diff
- let engine = Quillmark::new();
- let quill  = engine.quill_from_path(path)?;     // or engine.quill(tree)
- let result = quill.render(&doc, &opts)?;
- let formats = quill.supported_formats();
+ let quill  = quillmark::quill_from_path(path)?;  // or Quill::from_tree(tree)
+ let engine = Quillmark::new();
+ let result = engine.render(&quill, &doc, &opts)?;
+ let formats = engine.supported_formats(&quill)?;  // now returns Result
```

- `Quillmark::quill` / `quill_from_path` methods are removed. Use the core
  constructor `Quill::from_tree(tree) -> Result<Quill, Vec<Diagnostic>>`, or the
  `quillmark::quill_from_path` free function for filesystem loading (which
  surfaces a `RenderError`; fs stays out of core).
- `render`, `open`, `supported_formats`, and the new `supports_canvas` are now
  methods on `Quillmark`, each taking `&quill`. `supported_formats` returns
  `Result<_, RenderError>` (it resolves the backend).
- `validate`, `compile_data`, `dry_run`, `seed_*`, `backend_id` stay on `Quill`.

### `QuillSource` is gone; `Backend::open` takes `&Quill`

`QuillSource` was renamed to `Quill` and absorbed the wrapper's methods; the
orchestration `Quill` is deleted. Backends now receive `&Quill`:

```diff
  impl Backend for MyBackend {
-     fn open(&self, plate: &str, source: &QuillSource, json: &serde_json::Value)
+     fn open(&self, plate: &str, source: &Quill, json: &serde_json::Value)
          -> Result<RenderSession, RenderError> { … }
+
+     // Optional: override if your backend can paint to a canvas.
+     fn supports_canvas(&self) -> bool { false }
  }
```

If you reached the inner data via `quill.source()`, drop the call —
`quill.config()` / `quill.metadata()` / `quill.plate()` are now direct on
`Quill`.

## Python

The Python surface is unchanged: `Quillmark().quill_from_path(path)` still
returns a `Quill` with `render` / `open` / `supported_formats` /
`supports_canvas` / `metadata` / `validate` / `seed_*`. The engine pairing is
handled internally. One new failure mode: `metadata` and `supported_formats`
resolve the backend, so they raise `QuillmarkError` (`UnsupportedBackend`) for a
quill whose backend isn't registered — unreachable in the stock build, which
always registers Typst.

## `@quillmark/quiver`

### Coordinated release

`@quillmark/quiver` must be updated alongside this WASM release. A consumer who
updates `@quillmark/wasm` to `>=0.90.0` must also update `@quillmark/quiver` to
`>=0.14.0` at the same time — the older Quiver still calls the removed
`engine.quill(tree)` method.

### Code that bypassed Quiver should migrate to `getQuill`, not `fromTree`

Code that previously called `engine.quill(tree)` directly — bypassing a
`Quiver` instance — should migrate to `quiver.getQuill(ref)`, **not** to a
manual `loadTree` + `Quill.fromTree` sequence:

```ts
// wrong — duplicates Quiver's caching and construction
const tree = await quiver.loadTree(name, version);
const quill = Quill.fromTree(tree);

// right — Quiver handles resolution, construction, and caching
const quill = await quiver.getQuill(ref);
```

Holding a `QuillClass` reference (e.g. storing `wasmModule.Quill` on a
service field) is only needed when constructing quills **outside** of a
Quiver context (e.g. a raw-tree endpoint, a test fixture). Inside a Quiver
consumer, `getQuill` is the entire entry point.

### Quiver cache semantics changed

Under 0.89, `Quiver.getQuill(ref, { engine })` produced a separate `Quill`
instance per engine. Under 0.90 there is **one `Quill` per canonical ref** for
the lifetime of the `Quiver` instance, shared across every engine that calls
`getQuill`. The tree is fetched once, `Quill.fromTree` is called once, and the
result is cached. This halves the number of instances in multi-engine
deployments (e.g. a live-preview engine and a thumbnail-worker engine on the
same page).

Tests that asserted per-engine isolation behavior (e.g. "distinct engines get
distinct quill objects") should be removed; the shared-instance guarantee is the
new invariant.

### Quiver uses the core build; render path needs `getTree`

`@quillmark/quiver@0.14.0` materializes quills from `@quillmark/wasm/core`.
`getQuill(ref)` returns a core-memory `Quill` — correct for schema inspection,
validation, and blueprint access, but **not usable with the render engine**
(the two builds have separate WASM linear memories).

Code that loads a quill via Quiver and then passes it to `engine.render` must
cross the memory boundary explicitly. Use `quiver.getTree(ref)` to retrieve the
cached raw tree bytes and re-materialize a render-memory `Quill`:

```ts
// schema / validation — core quill, no engine needed
const coreQuill = await quiver.getQuill(ref);
coreQuill.schema;         // ✓
coreQuill.validate(doc);  // ✓

// rendering — re-materialize in render memory
import { Quill, Document, Quillmark } from "@quillmark/wasm";
const engine = new Quillmark();
const tree = await quiver.getTree(ref);
const renderQuill = Quill.fromTree(tree);
const renderDoc = Document.fromJson(coreDoc.toJson()); // round-trip
const result = engine.render(renderQuill, renderDoc, { format: "pdf" });
```

`getTree` reuses the same in-memory cache as `getQuill` — no extra network
fetch is paid. The tree is **not** evicted after `getQuill` resolves precisely
so `getTree` can return it without I/O.
