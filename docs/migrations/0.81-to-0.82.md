# 0.81.0 → 0.82.0 — card-yaml syntax

`0.82.0` replaces Quillmark's Markdown metadata syntax with the **card-yaml**
format. This is a **hard cutover**: the `0.81.0` `---`/`QUILL:` frontmatter and
the ` ```card <kind> ` fenced cards (and the legacy `---`/`CARD:` fence) are
removed and no longer parse. Stored documents must be migrated.

Alongside the syntax change, the structured-metadata type is **renamed**
`Frontmatter` → `Payload`, the card-API surface is reworked (see §6), and the
plate wire format changes shape (see §6.3). The rest of the API — `Document`,
`Quill`, and the rendering entry points — is unchanged.

## TL;DR

Every card is a `~~~card-yaml` fence. Each card begins with a **system-metadata
header** of `#@key: value` lines; the main card (the first card) declares
`#@quill` and `#@kind: main`:

```diff
- ---
- QUILL: my_quill
- title: Main Document
- ---
+ ~~~card-yaml
+ #@quill: my_quill
+ #@kind: main
+ title: Main Document
+ ~~~

  Some content here.

- ```card products
- name: Widget
- price: 19.99
- ```
+ ~~~card-yaml
+ #@kind: products
+ name: Widget
+ price: 19.99
+ ~~~

  Widget description.
```

Why: isolating the structured payload inside an explicitly delimited fenced
region — separate from the prose body that follows it — keeps LLM generation
stable and prevents state corruption. A generator editing prose cannot disturb
the structured fields, and vice versa.

## 1. The card-yaml card

A card has four parts, in order:

1. **Opening fence** — exactly `~~~card-yaml` (three tildes, at column zero).
   The info string alone identifies a card.
2. **System-metadata header** — a leading run of `#@key: value` lines inside
   the fence.
3. **Data payload** — standard YAML key/value pairs below the `#@` header.
4. **Closing fence** — exactly `~~~`.

The prose body begins immediately after the closing `~~~` and runs to the next
card or end of document.

### The main card and other cards

The document is **positional**: the **first** `~~~card-yaml` card is the
**main card**, and every later card is an ordinary card. "Main" is a positional
role, not a separate type — the main card has the same anatomy as every other
card. The main card must declare `#@kind: main`; `main` is a **reserved kind**
that no other card may use.

### System metadata (`#@`)

A card may begin with a run of `#@key: value` lines. The `#@` prefix keeps
these lines out of the YAML payload's user field set (a bare `#` line is an
ordinary YAML comment). System metadata carries no parser semantics beyond the
rules below.

- **`#@quill: <name>@<version>`** — binds the document to a quill (replacing
  the old `QUILL:` frontmatter key). The **main card must declare `#@quill`**,
  and it is exclusive to the main card.
- **`#@kind: <kind>`** — names a card's kind. This replaces the
  ` ```card <kind> ` info string and the legacy `CARD:` key. The **main card
  must declare exactly `#@kind: main`**; `main` is reserved and no other card
  may use it.
- **`#@id: <value>`** — an opaque, optional identifier. Plain metadata: no
  validation, no uniqueness; carried through the round-trip unchanged.

`#@` header lines may appear in **any order**; the emitter writes them in the
canonical order `quill`, `kind`, `id`. A duplicate `#@key` within a card, or a
malformed `#@` line, is a parse error.

### Blank-line rule

A blank line is required immediately above every `~~~card-yaml` opener, except
when the opener is the very first line of the document. A `~~~card-yaml` line
without a blank line above it is treated as an ordinary CommonMark fenced code
block, not a card (a `parse::card_fence_missing_blank` warning is emitted).

## 2. Migrating stored documents

For each document:

1. Replace the opening `---` of the frontmatter with `~~~card-yaml`, rename the
   `QUILL:` line to `#@quill:`, add a `#@kind: main` line below it, and replace
   the closing `---` with `~~~`.
2. For each card written as ` ```card <kind> `, replace the opener with
   `~~~card-yaml` followed by a `#@kind: <kind>` line, and replace the closing
   ` ``` ` with `~~~`.
3. For each legacy card written as a `---` fence with a `CARD:` key, replace
   the fences with `~~~card-yaml` / `~~~` and rename `CARD:` to `#@kind:`.
4. Ensure every `~~~card-yaml` opener has a blank line above it.

A bare `---` in body prose was already an ordinary CommonMark thematic break;
it is unaffected and needs no change.

`Document::to_markdown()` (`doc.toMarkdown()` in WASM, `doc.to_markdown()` in
Python) emits the **canonical card-yaml form**, so re-serializing a parsed
document produces compliant output automatically.

## 3. What is unchanged

- **Field names** still match `[a-z_][a-z0-9_]*`; **card kinds** still match
  `[a-z_][a-z0-9_]*`.
- `QUILL`, `CARD`, `BODY`, and `CARDS` remain **reserved** field names.
- YAML payload **comments** (own-line and inline `field: value  # note`) and
  the **`!fill`** placeholder tag are still supported and still round-trip.
- `Document`, `Quill`, and the rendering APIs keep their names and shapes
  across Rust, WASM, and Python. (The `Frontmatter` type is renamed, the
  `Card` metadata accessors are reworked, and the plate wire format changes
  shape — see §6.)

## 4. What is removed

- The `---`/`QUILL:` frontmatter.
- The ` ```card <kind> ` fenced-card syntax and the legacy `---`/`CARD:` card
  fence.
- **Comments on a `#@` header line.** The old `QUILL: ref  # note` inline
  comment has no equivalent — a `#@` header line carries no trailing comment.
  Move such notes to an own-line comment in the payload.
- The "cannot specify both QUILL and CARD" error — it cannot arise under
  card-yaml.

## 5. Error messages

Parse errors are reworded for the new syntax. Match on `ParseError` variants
(or diagnostic codes) rather than message text:

| Situation | Variant |
|---|---|
| No `~~~card-yaml` card, or the main card has no `#@quill` | `MissingQuill` |
| Main card missing `#@kind: main`, or declaring a different `#@kind` | `InvalidStructure` |
| A non-main card declaring `#@quill` or `#@kind: main` | `InvalidStructure` |
| `~~~card-yaml` opener with no `~~~` closer | `InvalidStructure` |
| Malformed `#@` header line, or a duplicate `#@key` within a card | `InvalidStructure` |
| Invalid `#@kind` value | `InvalidStructure` |
| Reserved name used as a field | `InvalidStructure` |
| Invalid YAML payload | `YamlErrorWithLocation` |

## 6. API changes

### 6.1 `Frontmatter` → `Payload` rename

The card-yaml format calls the structured YAML data below a card's `#@`
metadata header its **payload**. The in-memory type that held it —
historically named `Frontmatter` — is renamed to match. This is a pure rename:
the type's shape, methods, and behavior are unchanged.

**Rust** (`quillmark-core`):

| Old | New |
|---|---|
| `Frontmatter` | `Payload` |
| `FrontmatterItem` | `PayloadItem` |
| `Card::frontmatter()` | `Card::payload()` |
| `Card::frontmatter_mut()` | `Card::payload_mut()` |
| `RenderError::InvalidFrontmatter` | `RenderError::InvalidPayload` |
| `QuillConfig::coerce_frontmatter()` | `QuillConfig::coerce_payload()` |

**WASM** (`@quillmark/wasm`):

| Old | New |
|---|---|
| `card.frontmatter` | `card.payload` |
| `card.frontmatterItems` | `card.payloadItems` |
| `FrontmatterItem` (TS type) | `PayloadItem` |

**Python** (`quillmark`):

| Old | New |
|---|---|
| `doc.frontmatter` | `doc.payload` |
| `card["frontmatter"]` | `card["payload"]` |
| `card["frontmatter_items"]` | `card["payload_items"]` |

### 6.2 System metadata: `Sentinel` removed, `CardMetadata` added

A card's leading `#@key: value` lines are now **system metadata** drawn from
a closed set of three keys — `#@quill`, `#@kind`, `#@id`. Any other `#@key`
is a parse error. The API reflects this:

- The `Sentinel` enum is **removed**.
- A new type **`CardMetadata`** is exported — a typed struct with three
  optional public fields: `quill: Option<QuillReference>`, `kind:
  Option<String>`, `id: Option<String>`. `#@quill` is parsed into a typed
  `QuillReference` at parse time.
- `Card::sentinel()` is replaced by:
  - `Card::meta()` — the card's `CardMetadata` struct.
  - `Card::kind()` — the `#@kind` value, if present.
  - `Card::id()` — the `#@id` value, if present.
- `Card::is_main()` is **removed**, along with the `is_main` argument to
  `Card::from_parts`. Main vs. other cards is positional: the main `Card`
  lives in `Document::main` (and declares `#@kind: main`), the rest in
  `Document::cards`.
- `EditError::ReservedKind` is returned when a card kind of `main` is
  requested — `Card::new("main")` and `Document::set_card_kind(_, "main")`
  reject it, since `main` is reserved for the main card.

Code that matched on `Sentinel` variants or called `Card::sentinel()` must be
updated to read `meta()` / `kind()` / `id()`.

**Bindings.** The card object's discriminator key is renamed `tag` → `kind`
to match core vocabulary:

| Old | New |
|---|---|
| `card.tag` (WASM) / `card["tag"]` (Python) | `card.kind` / `card["kind"]` |
| `card.sentinel` / `card["sentinel"]` | **removed** — read `Document.main` vs `Document.cards` |
| `CardInput` `{ tag, fields?, body? }` | `{ kind, fields?, body? }` |

`Document.pushCard` / `insert_card` callers must pass `kind` instead of `tag`.

The same `tag` → `kind` vocabulary change renames the editing API in both
core and the bindings:

| Old | New |
|---|---|
| `Document::set_card_tag` / `setCardTag` (WASM) / `set_card_tag` (Python) | `set_card_kind` / `setCardKind` |
| `EditError::InvalidTagName` | `EditError::InvalidKindName` |
| `form::unknown_card_tag` diagnostic code | `form::unknown_card_kind` |

### 6.3 Plate wire format

`Document::to_plate_json()` — the JSON handed to backend plates — changes
shape. The flat form is replaced by a two-key object in which the main card
and every other card share the **same** card-object shape:

| Old | New |
|---|---|
| `{ "QUILL": ..., "<field>": ..., "BODY": ..., "CARDS": [...] }` | `{ "main": {...}, "cards": [...] }` |

The `main` object is `{ "QUILL": <ref>, "CARD": "main", <fields>, "BODY": ... }`;
each entry of `cards` is `{ "CARD": <kind>, <fields>, "BODY": ... }`.

Typst plates that read `data.<field>` / `data.BODY` / `data.CARDS` must move to
`data.main.<field>` / `data.main.BODY` / `data.cards`. Card-loop internals
(`card.CARD`, `card.BODY`, `card.<field>`) are unchanged.

`QuillConfig::schema()` correspondingly prepends a `CARD` (`const: "main"`)
discriminator field to `main.fields`, alongside the existing `QUILL` entry.
